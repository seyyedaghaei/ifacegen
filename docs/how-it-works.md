## How it works

`ifacegen` generates Go interfaces by analyzing your packages’ AST and type information, then emitting a `iface_gen.go` file per package.

### 1) Load packages

The CLI uses `golang.org/x/tools/go/packages` to load packages from the patterns you pass (for example `./...`).
It requests enough data to:

- parse syntax (AST)
- resolve types and method sets
- discover imports

### 2) Decide which structs to target

Within each package, `ifacegen` scans type declarations to decide which structs should produce interfaces:

- **Name matching** via `-match` (comma-separated globs like `*Service,*Repository`)
- **Explicit opt-in** via `// ifacegen:generate`
- **Explicit skip** via `// ifacegen:skip`

### 3) Collect methods (type-correct)

For each included struct type, `ifacegen`:

- uses `go/types` information to find the struct’s method set
- renders method signatures with correct parameter/return types
- carries over doc comments for the struct and methods when present

Because this uses the type checker, method signatures are accurate even with:

- imported types
- pointers vs values
- aliases and named types

### 4) Build imports deterministically

When a method signature references a type from another package, `ifacegen` adds an import for that package.
If multiple packages share the same import name, `ifacegen` automatically aliases them to avoid collisions.

### 5) Render and format output

`ifacegen` renders a template that starts with:

- `// Code generated by ifacegen. DO NOT EDIT.`

Then it formats the file using Go formatting tooling so it is gofmt-compliant.

### 6) Write only if changed

To keep diffs clean and avoid touching files unnecessarily, the CLI compares the newly generated bytes with the existing `iface_gen.go`.
If the content is unchanged, it does not rewrite the file.

### Extra modes

- **Dry-run** (`-dry-run`): computes output and prints a preview without writing files.
- **Interactive** (`-interactive`): prompts for options and prints an equivalent command.
- **Watch** (`-watch`): periodically checks for Go file changes and regenerates.
